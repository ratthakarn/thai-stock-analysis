## 🎯 เป้าหมาย

สร้าง **Stock Analysis Webapp สำหรับตลาดหุ้นไทย (SET)** มี 11 features, AI dual-model (Claude Haiku 4.5 + Sonnet 4.6), deploy บน Vercel + เก็บ cache บน Filebase. เน้น **cost optimization**: คำนวณ indicators/scoring ทั้งหมดใน JavaScript ใช้ AI เฉพาะขั้น "สรุป/วิเคราะห์" เท่านั้น. **ทุกหน้าต้องมี disclaimer ภาษาไทยว่าไม่ใช่คำแนะนำการลงทุน**

## 🧩 11 Features

1. **Dashboard** — index cards SET/mai/SETHD, market breadth (หุ้นขึ้น/ลง/นิ่ง), top gainers/losers, watchlist widget
2. **Stock Analysis** — candlestick chart + MA(20/50/200) + RSI + MACD + Bollinger Bands + Support/Resistance + fundamental (P/E, P/BV, div yield, ROE)
3. **Multi-Agent AI** — 3 Haiku agents (เทคนิค/ข่าว/fund flow) ขนานกัน → Sonnet CIO สังเคราะห์เป็น verdict (STRONG_BUY..STRONG_SELL) + buy/sell zone + target + stop loss
4. **News & Sentiment** — Google News RSS (ภาษาไทย) + Haiku สรุป + bullish/neutral/bearish score
5. **Watchlist** — เพิ่ม/ลบหุ้น, ราคา+%เปลี่ยน, เก็บ localStorage
6. **Price Alerts** — target บน/ล่าง, เช็คทุก 5 นาที, Web Notification API, localStorage
7. **Portfolio Tracker** — บันทึกหุ้นที่ถือ, P&L รายตัว+รวม
8. **PDF Export** — jsPDF + html2canvas ครอบคลุม ราคา/กราฟ/technical/AI verdict
9. **Fund Flow Report** — net buy/sell 4 กลุ่ม (ต่างชาติ/สถาบัน/บัญชีบริษัท/รายย่อย), bar chart 30 วัน, Haiku สรุป trend
10. **Volume & Order Trend** — OBV, Volume Delta, Buy/Sell Pressure % รายวัน
11. **Smart Screener** — สแกนหุ้น SET ทั้งตลาด, scoring 4 มิติ (Technical 0-3 + Volume 0-3 + Fund Flow 0-3 + Dividend 0-1 = เต็ม 10), แสดง top 20 + badge 🔥 Strong Buy / ✅ Watch + Haiku สรุปแต่ละตัว (cache 1 ชม.)

## 🛠 Tech Stack

- **Frontend:** React 18 + TypeScript + Vite + Tailwind CSS + Recharts + Zustand (UI state) + TanStack Query (server state)
- **Backend:** Express.js + TypeScript (รันด้วย `tsx`) — โครงสร้างต้องรันได้ทั้ง local และ Vercel serverless
- **Data:** TradingView scanner (หลัก, snapshot/screener/fundamental) + yahoo-finance2 (OHLCV history สำหรับ candlestick) + Google News RSS (rss-parser) + SET investor-type (Playwright scraper)
- **AI:** `@anthropic-ai/sdk` — Haiku `claude-haiku-4-5-20251001` (agents/summaries), Sonnet `claude-sonnet-4-6` (CIO synthesis) + prompt caching (`cache_control: ephemeral` บน system prompt)
- **Indicators:** `technicalindicators` npm (RSI, MACD, Bollinger, SMA, OBV)
- **Cache/Storage:** Filebase (S3-compatible) ผ่าน `@aws-sdk/client-s3` + localStorage (watchlist/portfolio/alerts)
- **Export:** jsPDF + html2canvas | **Concurrency:** p-limit | **Dev:** concurrently

## 🏗 Architecture

```
Local dev:   React(Vite :5173) --proxy /api--> Express(:3001)
Production:  React static(dist/) --/api/*--> Vercel serverless (api/index.ts = Express app)
Both backends call: TradingView scanner | yahoo-finance2 | Google News RSS | Anthropic | Filebase
```
- แยก `server/app.ts` (สร้าง+return Express app, **ไม่มี** `.listen()`) ออกจาก `server/index.ts` (local: `import app; app.listen()`)
- `api/index.ts` (Vercel): `import app from server/app; export default app`
- Filebase ใช้เก็บ `screener-results.json` + `fundflow-history.json` (Vercel serverless เป็น ephemeral → in-memory cache หาย ต้องใช้ object storage)

## 📦 ลำดับการสร้าง (8 phases)

1. **Foundation** — scaffold Vite+React+Tailwind+Express, `server/services/yahoo.ts`, proxy, types, page stubs + routing + sidebar + disclaimer
2. **Dashboard + Stock Analysis** — TradingView scanner service, candlestick chart, indicators util, RSI/MACD charts
3. **AI Multi-Agent** — `claude.ts` (dual-model + prompt cache), `/api/ai/analyze` (3 Haiku ขนาน → Sonnet CIO)
4. **News + Watchlist + Alerts** — RSS + Haiku sentiment, Zustand+persist stores, alert checker hook
5. **Portfolio + PDF Export**
6. **Fund Flow + Volume** — SET scraper (Playwright) + Filebase, volume indicators
7. **Smart Screener** — scoring service + Filebase cache + lazy Haiku summary
8. **Polish** — disclaimer ทุกหน้า, error handling ภาษาไทย, responsive, loading states

## ⚠️ กับดักที่เจอจริง + วิธีแก้ (อ่านให้ครบ! จะประหยัดเวลา debug หลายชั่วโมง)

1. **yahoo-finance2 v3 ต้อง instantiate:** `const yf = new YahooFinance({ suppressNotices: ['yahooSurvey'] })` — ถ้าเรียก static แบบ v2 จะ error "Call new YahooFinance() first"

2. **SET index symbols ต้องลงท้าย `.BK`:** ใช้ `^SET.BK`, **`^MAI.BK`**, **`^SETHD.BK`** — `^MAI`/`^SETHD` เฉยๆ Yahoo คืน `undefined` → โค้ดอ่าน property ของ undefined → **500**. ใส่ guard `if (!result) throw` ใน getQuote ด้วย

3. **ใช้ `API_PORT` ไม่ใช่ `PORT`:** dev harness/preview บางตัว inject `PORT=5173` ทำให้ Express ไปยึด port เดียวกับ Vite. ตั้ง backend เป็น `process.env.API_PORT ?? 3001`

4. **TradingView scanner = แหล่งข้อมูลหลัก (ฟรี, ไม่ต้อง key):**
   - `POST https://scanner.tradingview.com/thailand/scan` พร้อม header `User-Agent` + `Origin/Referer: tradingview.com`
   - body: `{columns:[...], filter:[{left:"type",operation:"equal",right:"stock"}], range:[0,1000], sort:{...}}`
   - columns ที่ใช้: `name,description,close,change,change_abs,volume,average_volume_10d_calc,relative_volume_10d_calc,RSI,MACD.macd,MACD.signal,SMA20,SMA50,SMA200,dividends_yield,price_earnings_ttm,market_cap_basic`
   - response: `{data:[{s:"SET:PTT", d:[...]}]}` — symbol มาเป็น `SET:PTT`, **กรองตัวลงท้าย `.R` ออก** (foreign board ซ้ำ)
   - 1 request ได้หุ้นทั้งตลาด → screener/movers/breadth เร็วมาก ไม่ต้องยิงทีละตัว. แต่ **ให้ history ไม่ได้** → candlestick ใช้ Yahoo
   - เป็น unofficial endpoint → wrap try/catch + fallback ไป Yahoo เสมอ

5. **SET fund flow อยู่หลัง Incapsula (บล็อก datacenter IP):** scrape ตรงๆ ได้แต่หน้า challenge. วิธีที่ใช้ได้: **Playwright เปิด browser จริงบนเครื่อง local** (residential IP) → `page.goto` หน้า investor-type (ผ่าน Incapsula + เซ็ต cookie) → `page.evaluate(() => fetch('/api/set/market/SET/investor-type-chart?period=5D').then(r=>r.json()))` ยิง internal API จาก *ใน browser context*
   - `period=5D` = รายวัน (5 จุด), `period=1Y` = รายเดือน (~13 จุด) — period อื่น (1M/3M/YTD) คืน 400
   - ค่าหน่วยเป็นบาท → หารด้วย 1e6 เป็นล้านบาท. กลุ่ม `individual` = รายย่อย (retail)
   - scraper merge แบบ rolling 30 วัน (dedupe by date, keep last 30) → idempotent รันซ้ำได้
   - รันบน Vercel serverless ไม่ได้ (headless browser + datacenter IP โดนบล็อก) → รัน local แล้วเขียนผลลง Filebase, เว็บอ่านจาก Filebase

6. **Filebase setup:** `new S3Client({ endpoint:'https://s3.filebase.com', region:'us-east-1', credentials:{...}, forcePathStyle:true })`. ทำ in-memory Map fallback เมื่อไม่มี creds (local dev ทำงานได้โดยไม่ต้องมี Filebase)

7. **Vercel SPA fallback (สำคัญ!):** `vercel.json` ต้องมี rewrite catch-all ไป `index.html` ไม่งั้นเปิด/refresh sub-route (`/screener` ฯลฯ) จะ **404**:
   ```json
   "rewrites": [
     { "source": "/api/(.*)", "destination": "/api/index" },
     { "source": "/(.*)", "destination": "/index.html" }
   ]
   ```
   (local vite dev ไม่เจอบั๊กนี้เพราะมี fallback ในตัว — เจอเฉพาะบน Vercel)

8. **Vercel deploy ถูก BLOCKED:** ถ้า git `user.email` ไม่ใช่ email จริง (เช่น default `user@hostname.local`) Vercel จะ block ทุก deploy. ต้อง `git config --global user.email "<email ตรงกับ GitHub account>"` ก่อน commit แรก

9. **technicalindicators alignment:** ผลลัพธ์ array สั้นกว่า input ตาม period (เช่น RSI(14) สั้นกว่า 14 ตัว) → pad `null` ด้านหน้าให้ align กับ OHLCV ก่อน merge ลง chart data

10. **jsPDF ไม่รองรับฟอนต์ไทย:** ใช้ **html2canvas** rasterize DOM เป็นภาพก่อน แล้วใส่ภาพลง PDF (ข้อความไทยกลายเป็น pixel เลยไม่ต้อง embed ฟอนต์)

11. **Fund flow route ต้องอ่านอย่างเดียว:** route อ่าน Filebase แล้ว serve เลย **ห้ามเขียน mock ทับตาม TTL** (ไม่งั้นจะทับข้อมูลจริงที่ scraper เขียนไว้). ใช้ mock เฉพาะตอน Filebase ว่างเปล่าจริงๆ (bootstrap). อย่าตั้ง Vercel cron เขียน fundflow (มันรัน mock บน datacenter)

12. **Claude response parsing:** บังคับ output เป็น JSON ใน system prompt + เขียน `parseJSON()` ที่ทน prose/fences (regex จับ `{...}` เป็น fallback)

## 🚀 Deployment + Scheduling

- **Deploy:** push GitHub → import ใน Vercel → ตั้ง env vars (`ANTHROPIC_API_KEY`, `FILEBASE_KEY/SECRET/BUCKET`, `CRON_SECRET`) → Deploy. แก้ภายหลัง = `git push` → auto-deploy
- **Screener:** รัน on-demand ได้ (TradingView scan + scoring < 60s) + Filebase cache 1 ชม. + Vercel cron วันละ 1 ครั้ง (Hobby limit)
- **Fund flow scraper:** ต้องรันบนเครื่อง local (residential IP). ตั้ง cron + launchd (macOS) ให้รันหลังตลาดปิด (~18:00) เขียนผลลง Filebase
- **`.env.example`** บอก env ที่ต้องมี + แหล่งที่มา; secrets backup อยู่ที่ Vercel (`vercel env pull .env` กู้ได้)
